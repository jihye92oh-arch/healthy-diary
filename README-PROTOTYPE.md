# Healthy Diary - 바닐라 JavaScript 프로토타입

건강한 다이어트 습관 형성을 돕는 개인 맞춤형 건강 관리 플랫폼의 바닐라 JavaScript 프로토타입입니다.

## 🚀 실행 방법

### 간단한 실행
1. `vanilla-prototype/index.html` 파일을 더블클릭하여 브라우저에서 엽니다
2. 또는 브라우저에서 `index.html` 파일을 드래그 앤 드롭합니다

### 로컬 서버 실행 (권장)
```bash
# Python 3가 설치되어 있다면
cd vanilla-prototype
python -m http.server 8000

# 또는 Python 2
python -m SimpleHTTPServer 8000

# Node.js가 설치되어 있다면
npx serve
```

그 후 브라우저에서 `http://localhost:8000` 접속

## 📋 주요 기능

### 1. 목표 설정
- 일일 목표 칼로리 설정 (500~5000 kcal)
- 목표는 로컬 스토리지에 저장되어 브라우저 재시작 후에도 유지됩니다

### 2. 식단 기록
- **식사 유형 선택**: 아침, 점심, 저녁, 간식
- **음식명 입력**: 자유 텍스트 입력
- **칼로리 입력**: 수동 입력 (1~2000 kcal)
- **자동 계산**: 총 섭취 칼로리 자동 합산
- **기록 삭제**: 개별 항목 삭제 가능

### 3. 운동 기록
- **운동명 입력**: 자유 텍스트 입력 (예: "조깅 30분", "요가 1시간")
- **소모 칼로리 입력**: 수동 입력 (1~2000 kcal)
- **자동 계산**: 총 소모 칼로리 자동 합산
- **기록 삭제**: 개별 항목 삭제 가능

### 4. 진행률 대시보드
- **섭취 칼로리**: 식단 기록의 총합
- **소모 칼로리**: 운동 기록의 총합
- **남은 칼로리**: `목표 - 섭취 + 소모` 계산
- **진행률 바**: 목표 대비 퍼센트 시각화
  - 0-79%: 초록색 (정상)
  - 80-99%: 주황색 (주의)
  - 100% 이상: 빨간색 (목표 초과)

### 5. 데이터 지속성
- **로컬 스토리지 활용**: 모든 데이터는 브라우저에 저장
- **자동 저장**: 모든 작업 후 자동으로 저장
- **데이터 유지**: 브라우저를 닫았다 열어도 데이터 유지
- **초기화 기능**: 모든 데이터 삭제 가능 (되돌릴 수 없음)

## 🛠️ 기술 스택

- **HTML5**: Semantic HTML, 접근성 고려
- **CSS3**: Flexbox, Grid, CSS Variables, 반응형 디자인
- **Vanilla JavaScript (ES6+)**: 프레임워크 없이 순수 JavaScript
- **로컬 스토리지**: 클라이언트 사이드 데이터 저장

## 📱 반응형 디자인

- **데스크탑** (1200px+): 2열 그리드 레이아웃
- **태블릿** (768px-1199px): 적응형 레이아웃
- **모바일** (767px 이하): 1열 스택 레이아웃

모든 디바이스에서 최적화된 경험을 제공합니다.

## 🔒 보안 및 데이터 처리

### 데이터 처리 원칙
- **칼로리 계산**: 모든 계산에서 `Math.round()` 사용 (반올림)
- **소수점 버림 금지**: `Math.floor()` 절대 사용 안 함
- **입력 검증**: 모든 사용자 입력 검증
- **XSS 방지**: `textContent` 사용 (innerHTML 사용 안 함)

### 데이터 저장
- **로컬 전용**: 모든 데이터는 브라우저의 로컬 스토리지에만 저장
- **서버 전송 없음**: 백엔드가 없어 데이터가 외부로 전송되지 않음
- **개인정보 보호**: 사용자 데이터는 브라우저 내에서만 존재

## 📊 사용 예시

### 시나리오: 하루 식단 및 운동 기록

1. **목표 설정**: 2000 kcal

2. **아침 식사 기록**:
   - 식사 유형: 아침
   - 음식명: 현미밥
   - 칼로리: 300 kcal

3. **점심 식사 기록**:
   - 식사 유형: 점심
   - 음식명: 닭가슴살 샐러드
   - 칼로리: 450 kcal

4. **간식 기록**:
   - 식사 유형: 간식
   - 음식명: 사과
   - 칼로리: 52 kcal

5. **운동 기록**:
   - 운동명: 조깅 30분
   - 소모 칼로리: 250 kcal

**결과 대시보드**:
- 섭취 칼로리: 802 kcal (300 + 450 + 52)
- 소모 칼로리: 250 kcal
- 남은 칼로리: 1,448 kcal (2000 - 802 + 250)
- 진행률: 28% (정상 범위)

## ⚠️ 제한사항

이 프로토타입은 다음 기능이 **포함되지 않습니다**:

1. **백엔드 연동**: 서버 없이 클라이언트만 작동
2. **사용자 인증**: 로그인/회원가입 기능 없음
3. **음식 데이터베이스**: 음식 검색 및 자동완성 없음
4. **AI 추천**: Claude API를 통한 식단/운동 추천 없음
5. **날씨 연동**: 날씨 기반 운동 추천 없음
6. **차트/그래프**: 시각화 라이브러리 없음
7. **영양소 분석**: 단백질/탄수화물/지방 분석 없음
8. **계절별 추천**: 제철 식재료 추천 없음

이러한 기능들은 React 버전에서 구현될 예정입니다.

## 🔄 향후 React 마이그레이션 계획

이 바닐라 JavaScript 프로토타입은 다음과 같이 React로 마이그레이션될 예정입니다:

### 컴포넌트 구조
```
App
├── Header
├── GoalSetting
├── Dashboard
│   ├── StatCard (섭취/소모/남은 칼로리)
│   └── ProgressBar
├── DietForm
├── ExerciseForm
├── RecordList
│   ├── DietList
│   └── ExerciseList
└── Footer
```

### 기술 스택 추가
- **React 18**: 컴포넌트 기반 구조
- **TypeScript**: 타입 안전성
- **React Hook Form**: 폼 관리
- **React Query**: 데이터 fetching & 캐싱
- **Recharts**: 데이터 시각화
- **Tailwind CSS**: 유틸리티 퍼스트 스타일링

### 추가 기능
- 음식 데이터베이스 연동 (식품의약품안전처 API)
- AI 기반 식단/운동 추천 (Claude API)
- 날씨 기반 운동 추천 (OpenWeatherMap API)
- 주간/월간 통계 및 차트
- 영양소 균형 분석
